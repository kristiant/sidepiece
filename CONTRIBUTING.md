## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create a branch** for your feature or fix
4. **Make your changes**
5. **Test** thoroughly
6. **Submit a Pull Request**

## Development Setup

### Prerequisites
- macOS 13.0+ (Ventura or later)
- Xcode 15.0+
- Swift 5.9+

### Building
```bash
git clone https://github.com/YOUR_USERNAME/sidepiece.git
cd sidepiece
open Sidepiece.xcodeproj
```

## Code Style

- Follow Swift API Design Guidelines
- Use Australian English spelling in user-facing strings
- Prefer `guard` for early returns
- Use descriptive variable names
- Keep functions focused and small

## Commit Messages

Use clear, descriptive commit messages:

```
feat: add snippet category support
fix: resolve clipboard paste delay
docs: update installation instructions
refactor: extract hotkey handling into service
```

## Pull Requests

- Keep PRs focused on a single feature or fix
- Update documentation if needed
- Add tests for new functionality
- Ensure all existing tests pass

## Reporting Issues

When reporting bugs, please include:
- macOS version
- Steps to reproduce
- Expected vs actual behaviour
- Any relevant logs or screenshots

## Questions?

Feel free to open a Discussion or reach out via Issues.
